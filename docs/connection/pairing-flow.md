# Pairing & Connection Flow

## Overview

PocketDev uses a one-time setup code for initial device pairing, followed by Ed25519 keypair authentication for all subsequent requests.

## Pairing Sequence

```
┌─────────────┐                            ┌──────────────┐
│  Agent       │                            │  Mobile App  │
│  Server      │                            │              │
└──────┬───────┘                            └──────┬───────┘
       │                                           │
       │  1. Generate setup code (ABCD-1234)       │
       │     15-minute expiry                      │
       │                                           │
       │  2. Display QR code + URL in console      │
       │     pocketdev://192.168.1.1:4387/ABCD1234 │
       │                                           │
       │                    3. Scan QR or paste URL │
       │◄──────────────────────────────────────────│
       │     POST /PocketDev/pair                  │
       │     { code, publicKey, deviceName }       │
       │                                           │
       │  4. Validate code, register device        │
       │────────────────────────────────────────►  │
       │     { deviceId, serverPublicKey }         │
       │                                           │
       │  5. Setup mode disabled                   │
       │                                           │
       │                 6. Connect WebSocket       │
       │◄──────────────────────────────────────────│
       │     WS /PocketDev/ws                      │
       │     Auth: PocketDev <signed header>       │
       │                                           │
```

## Setup Code Format

Generated by agent server (`services/setup.ts`):

```
Format: ABCD-1234 (4 letters + hyphen + 4 digits)
Expiry: 15 minutes from generation
One-time use: Disabled after successful pairing
```

The code is displayed as:
- **QR code**: JSON payload `{ host, port, code }` rendered via qrcode library
- **Connection URL**: `pocketdev://host:port/code`
- **Manual display**: Host, port, and code shown separately

## Connection URL Protocol

```
pocketdev://192.168.1.1:4387/ABCD1234
└─scheme─┘ └────host────┘└port┘└─code─┘
```

Mobile app parses via regex: `/^pocketdev:\/\/([^:/]+):(\d+)\/(.+)$/i`

The code is normalized: uppercase, non-alphanumeric stripped, hyphen inserted at position 4.

## Device Registration

On successful pairing (`POST /PocketDev/pair`):

1. Server validates setup code matches and hasn't expired
2. Generates device ID (16 random bytes, hex-encoded)
3. Stores in `devices` table: `{ id, public_key, name, platform }`
4. Gets/creates server Ed25519 keypair (stored in `server_config` table)
5. Returns `{ deviceId, serverPublicKey }` to mobile
6. Disables setup mode (setup code cleared)

## Ed25519 Key Exchange

- **Mobile**: Generates keypair on first launch via `@noble/ed25519`
  - Private key stored in MMKV (encrypted at rest)
  - Public key sent during pairing
- **Server**: Generates keypair on first boot
  - Stored in `server_config` SQLite table
  - Public key returned during pairing

## Post-Pairing Authentication

All subsequent requests include a signed header:

```
Authorization: PocketDev <base64(JSON)>

JSON payload:
{
  deviceId: string,
  timestamp: number,      // Current time in ms
  signature: string       // Ed25519 signature of timestamp
}
```

**Validation** (server-side):
1. Parse header format
2. Decode base64 JSON
3. Check timestamp within 30-second window
4. Look up device by `deviceId` in database
5. Verify Ed25519 signature using stored public key
6. Update `lastSeen` timestamp

## ConnectScreen Flow

**Source**: `apps/mobile/src/screens/ConnectScreen.tsx`

### Fresh Pairing
1. User scans QR or pastes URL
2. URL parsed → `{ host, port, code }`
3. `pairWithServer(host, port, code)` called
4. On success: `setPaired({ ip, port, deviceId })`
5. Checks prerequisites → determines post-pair destination
6. Shows `PairingAnimation`
7. On animation complete: navigate to `ServerSetup` or `Main`

### Existing Server Reconnect
1. If `existingServer` in connection store → show reconnect banner
2. User taps reconnect → `connect()` (establishes WebSocket)
3. Checks prerequisites readiness
4. Navigate to `ServerSetup` (if not ready) or `Main` (if ready)

### Unpair
1. User taps "Unpair" button
2. `unpair()` called → disconnects WS, calls `DELETE /PocketDev/unpair`, clears storage
3. Remains on Connect screen for re-pairing

## Dev Mode

When `POCKETDEV_DEV_MODE=1`:
- Auto-registers a test device on first boot
- Auth validation bypassed for all requests
- Useful for local development without QR scanning
